# Troubleshooting Call — ECI (eAutomate)
**Date:** 2026-06-23
**Attendees:** Ralph Compton, David Kobs (RCL)
**With:** ECI Software Solutions (makers of eAutomate)

## Context

eAutomate is currently hosted on a virtual desktop as a Windows application. This creates friction for any automation that needs to read or write data — there is no clean programmatic access path today.

## Goal

Identify a REST API or alternative data access method that would allow the toner replenishment automation to interact with eAutomate without going through the virtual desktop UI.

## Questions to Resolve

- Does eAutomate expose a REST API? If so, what is covered (supply orders, equipment records, serial number lookup)?
- What authentication method does the API use?
- Are there rate limits or sandbox-specific restrictions?
- If no REST API, what other integration options exist (ODBC, web services, data export/sync)?
- What would be required to enable API access for Patriot Group's instance?

## Outcome

API access questions were not resolved on the call. No confirmed REST API or alternative data path identified yet.

## Post-Call Update (2026-06-24)

eAutomate exposes a SOAP API called **PIP**:
- Endpoint: `PublicAPIService.asmx`
- Serial number lookup: `getEquipmentFromSerialNumber` (single and batch variants)
- This covers the printer branch lookup (serial → Equipment ID)
- **API enablement for Patriot Group's instance is still required** — needs action with ECI or Chris Kobs
